Detailed explanation of calculation of ranges â€˜aâ€™, â€˜bâ€™, â€˜câ€™, and â€˜dâ€™ in Figure S.1

The calculation of ranges â€˜aâ€™ and â€˜bâ€™ in Fig. 1A is described with respect to the more detailed Fig. S.1A, which is calculated here:

res_vec <- create_intervals(rec_2021)

plot(res_vec,
     type = "eti",
     xlim = c(0, 40),
     ylim = c(0, 0.11),
     interval_arrows = TRUE,
     xlab = "Recruitment (billions of fish)",
     arrowhead_gap = 0.1,
     explanatory_lines_a_b = TRUE)

This shows the ETI in Fig. 1A but with extra lines. Two of the vertical grey lines show the recruitment values at the ends of the ETI (at the red/blue boundaries), denoting the right ends of â€˜aâ€™ and â€˜bâ€™. Horizontal grey lines show the densities at these two values; the intersection of these lines with the opposite limb of the distribution gives the other two vertical grey lines, defining the left ends of â€˜aâ€™ and â€˜bâ€™. Therefore, â€˜aâ€™ and â€˜bâ€™ both span the same range of densities on the y-axis, but â€˜aâ€™ is outside of the ETI while â€˜bâ€™ is inside of it. (See also Supporting Information).

Another undesirable feature of ETIs (for asymmetric distributions) is that there we can construct ranges c and d for which all values in c are more likely than those in d, yet c is outside the ETI and d is inside it:

plot(res_vec,
     type = "eti",
     xlim = c(0, 40),
     ylim = c(0, 0.11),
     interval_arrows = TRUE,
     xlab = "Recruitment (billions of fish)",
     arrowhead_gap = 0.1,
     explanatory_lines_a_b = FALSE,
     show_a_b = FALSE,
     explanatory_lines_extra = TRUE)

Here, the horizontal green line is set (as an example) halfway between the densities at the ends of the ETI (horizontal grey lines are as in the previous figure). Its intersections with the density curve define the left ends of c and d, and all values within c are more likely than all those in d, but c is outside the ETI and d is inside it. The green line can be placed at any point between the two grey lines to create such ranges c and d. This includes at the density corresponding the HDI, as in Figure 12.2 of Kruschke (2015; Doing Bayesian Data Analysis: A Tutorial with R, JAGS, and Stan. 2nd Edition).

The above plot settings are incorporated into our function figure_s1() for reproducing Fig. S1.

Computing the HDI on the kernel density estimate

Given the minor issue in Fig 1B that the densities at the endpoints of the HDI are not exactly equal, but they should be in theory, we also tried another approach to computing the HDI. This involved first calculating the kernel density estimate on the MCMC sample, and then using that to calculate the HDI. The HDInterval package includes the function to do this, which we incorporate into our create_intervals() function with the argument density = TRUE.

So repeating our Fig. 1B of 2021 hake recruitment, but using the density option, we obtain:

One technical issue we resolved was that the default of 512 equally spaced points (n) at which to compute the kernal density estimate in the density() function, did not resolve the issue of unequal densities at the endpoints of the HDI. Setting the number of points to 512 here shows this:

res_2021_n_512 <- create_intervals(rec_2021,
                                   n = 512)
plot(res_2021_n_512,
     xlim = c(0, 40),
     xlab = "Recruitment (billions of fish)")

We found that increasing n to 100,000 resolved this (this does not increase the wiggliness of the kernel density estimate, just the number of points at which it is calculated), and hence is set as the default in our create_intervals() function.

So while the density approach (with n=100,000) did seem preferable, given the equal densities at the endpoints of the HDI, the following simple simulation experiment convinced us to stick with the moving-window approach. We present these results for interested users.

Simulation experiment

We simulate 8,000 values from a lognormal distribution with mean on the log scale of 4 and standard deviation of 0.5, somewhat resembling our recruitment MCMC samples. We calculate the HDI using our default approach and with the density approach just described. We repeated this experiment 1,000 times, to compare the calculated HDIâ€™s with the known true theoretical values (because we know the true underlying distribution, unlike for our MCMC samples).

Multimodal distributions

Multimodal distributions of MCMC output can occur from the underlying stochasticity of MCMC results, and also from ensemble approaches whereby results from multiple models are combined to provide model-averaged result. To investigate the associated intervals, we first create a simple bimodal distribution of simulated data and calculate an HDI using the moving-window approach:

                        3.5,

As can be seen in this example, there is a region (around 17) within the HDI but with a low density, and the difference between the densities at the endpoints of the interval is quite different (though still less so than for the ETI). Thus, this is not strictly an HDI, but is still a valid credible interval. This is using the default moving-window approach. To obtain disjoint intervals (and therefore highest density regions, not intervals) requires using the density approach. Thus, there are two disjoint intervals that make up the highest density region. For stock assessments, it is unlikely to be helpful to report such disjoint intervals for giving advice, and so we would not recommend using them. Furthermore, in the simulations above we found the density approach to not be accurate (for the sample size we tested). Therefore we recommend investigating the use of HDIs or ETIs, noting that such HDIs are still valid credible intervals and will be shorter than the corresponding ETIs.
